## 0) Context

We are solving **ECG image digitization**:

- **Input:** ECG images (multiple “types” per ECG id)  
- **Output:** digitized signals (12 leads + rhythm) formatted for Kaggle submission

We already have a working segmentation pipeline (**Stage B**) that performs well on clean images (type `0001`) but collapses on the hardest domains (notably `0005`, `0010`, `0006`). Deterministic preprocessing/rectification helped but did not close the domain shift gap.

### Type definitions (from dataset)
Images are stored as: `train/[id]/[id]-[type].png`

| Type | Description |
|---|---|
| `0001` | Original color ECG image (generated by ECG-image-kit) |
| `0003` | Printed in color and scanned in color |
| `0004` | Printed in color and scanned in black & white |
| `0005` | Mobile photos of color printed images (table/background) |
| `0006` | Mobile photos of ECGs on a laptop screen |
| `0009` | Mobile photos of stained and soaked printed ECGs |
| `0010` | Mobile photos of printed ECGs with extensive damage |
| `0011` | Scans of printed ECG images with mold (color) |
| `0012` | Scans of printed ECG images with mold (black & white) |

## 1) High‑level Architecture

### Stage B (existing): Segmenter (image → masks → signals)
1. **Rectify + canonicalize** input image (Hough-based + perspective warp + scan-like gray)
2. **Segmentation U‑Net** predicts `K=13` mask/logit channels (leads layout)
3. Decode masks → centerlines → signals → submission CSV

Stage B model (already used in the project):

```python
import segmentation_models_pytorch as smp

stageB = smp.Unet(
    encoder_name="resnet34",
    encoder_weights="imagenet",
    in_channels=1,
    classes=K,   # K = 13
)
```

### Stage A (new): Scanifier / Cleaner (hard image → clean canonical image)
Stage A is an **image-to-image** model trained to convert hard domains into the same canonical representation that Stage B expects.

Inference becomes:

```
raw image
  -> preprocess_for_model (rectify/warp/scan-like gray)
  -> Stage A (scanifier)                 [NEW]
  -> Stage B (segmenter)                 [EXISTING]
  -> decode leads -> submission
```

## 2) Key Decisions (Decision Log)

This is the “why we did this” log for handover & reproducibility.

### Confirmed “not a bug”
- We already have a fallback mechanism (zeros/baseline) for decoding failures.
- px-per-mm calibration is not the primary bottleneck for the observed collapse.
- Multiple deterministic preprocessing styles were tried; they did not solve hard-domain failures.

### Why deterministic preprocessing alone is insufficient
Hard domains include artifacts that are hard to remove with fixed rules:
- monitor moiré patterns (`0006`)
- table/background texture + lighting (`0005`)
- torn / destroyed paper patterns (`0010`)
- mold/stains/soak/shadows (`0009`, `0011`, `0012`)

### Strategic pivot
**Add Stage A (learned scanifier)**:
- Input: hard types (0005/0010/0006/…)
- Target: canonicalized type `0001` (“clean” reference)
- Train Stage A with losses that prevent hallucination and preserve the waveform

### Why we keep Stage B
Stage B already works well on clean canonical `0001`. The objective is to **bring hard domains into the same distribution** rather than reinvent segmentation from scratch.

## 4) Stage A Training (Scanifier)

### 4.1 Data pairing
For each ECG id:

- **Target (`tgt`)**: canonicalized `0001`
- **Source (`src`)**: canonicalized hard type sampled from:
  `0004, 0005, 0006, 0009, 0010, 0011, 0012`

We oversample the worst types:
- `0005` (table/background) — very hard
- `0010` (extensive damage) — very hard
- `0006` (screen/moire) — hard

We also add **identity samples** with probability `p_identity ~ 0.10`:
- `src = tgt = canonical(0001)`
This prevents Stage A from “stylizing” already-clean scans.

**Note about `0003`:**
- Not catastrophic in our sample SNR, so it was initially excluded to focus capacity on worst types.
- You can add `0003` later as a second-phase curriculum to improve general normalization.

### 4.2 Stage A model
We use **two separate models** (one per stage). They can share architecture/backbone but have different heads:

```python
import segmentation_models_pytorch as smp

# Stage A (student): image -> image
stageA = smp.Unet(
    encoder_name="resnet34",
    encoder_weights="imagenet",
    in_channels=1,
    classes=1,          # 1-channel cleaned grayscale
    activation=None,    # apply sigmoid/clamp in training code
)
```

### 4.3 Teacher concept (Stage B frozen)
Stage B acts as a **teacher** during Stage A training only:

- Freeze Stage B parameters (`requires_grad=False`)
- Add a teacher-consistency loss so Stage A outputs images that Stage B can segment correctly

Teacher is **not** used during inference beyond normal Stage B segmentation.


### Freezing (weights and BatchNorm stats): theory and rationale

In this project we use “freezing” in two different senses, and both are motivated by **stability under domain shift**:

1) **Freeze the teacher (Stage B) weights** during Stage A training. Stage B defines the teacher-consistency target (what “useful cleaning” means). If Stage B were updated at the same time, the target would move every step (“moving target” problem), which makes optimization unstable and can degrade the already-good segmenter. Freezing Stage B gives Stage A a **fixed, reliable objective** and also reduces memory/compute (no gradients or optimizer state for Stage B). Note that gradients still flow **through** Stage B to Stage A’s output image (Stage B is differentiable w.r.t. its input), but Stage B’s parameters do not change.

2) **Freeze BatchNorm running statistics** (optional but often helpful), especially with small batch sizes and mixed domains (0001 vs 0005/0010/0006…). BatchNorm maintains running mean/variance buffers used at inference. When batches come from very different distributions, these running stats become a “bad average” that matches none of them, hurting generalization. Freezing BN stats prevents noisy/minibatch-specific updates and makes training behave closer to using **stable normalization**. This is particularly relevant for our setting because (a) batch sizes are small on GPU, and (b) domain shift is extreme.

Practical guideline: **always freeze Stage B weights** (teacher). Consider freezing BN running stats for Stage A if training is unstable or outputs fluctuate; unfreeze later if the model seems under-adapting.

### 4.4 Encoder initialization (copy, not reference)
Optionally initialize Stage A encoder from Stage B encoder:

```python
stageA.encoder.load_state_dict(stageB.encoder.state_dict(), strict=True)
```

This **copies values** (no shared references). Stage A and Stage B remain independent models.

### 4.5 Losses (recommended baseline)
Stage A loss is a weighted sum:

1) **Reconstruction:** Charbonnier (or L1) between `pred` and `tgt`  
2) **Gradient loss:** preserve thin waveform lines / reduce blur  
3) **Identity loss:** only on identity samples  
4) **Teacher consistency:** enforce `StageB(pred)` ≈ `StageB(tgt)` (probabilistic target)

Example total loss:
```
L = 1.0 * L_rec
  + 0.2 * L_grad
  + 0.5 * L_teacher
  + 0.1 * L_identity
```

### 4.5.1 Loss evolution (why we changed it)

We started with an **unweighted** objective (Charbonnier/L1 + gradient + teacher-consistency + identity). In practice, Stage A often **collapsed to a trivial solution**:

- **“Mostly white” outputs** (background dominates the image area).
- Stage B then predicts **near‑zero everywhere** ⇒ masks look **all black**.
- `ink_mean(StageB(tgt))` stayed around a small constant, while `ink_mean(StageB(StageA(src)))` dropped near zero.

**Why this happens (core issue):** ECG pages are mostly background. If losses treat every pixel equally, the model can reduce loss cheaply by matching background statistics and ignoring the thin waveform strokes. The teacher loss can also be dominated by background if we compare logits/probabilities uniformly over the full image.

#### The fix: waveform‑focused weighting (teacher‑derived weight map)

We switched to a **teacher‑derived weight map** `w` computed from the clean target `tgt`:

1) Run frozen teacher on the target: `p = sigmoid(StageB(tgt))`  
2) Convert to “inkness” map: `ink = max_k p_k`  
3) Threshold + (optional) dilate to cover thin strokes and small misalignments  
4) Boost waveform regions by a factor `alpha`

Then we apply that weight map to losses so the model is *forced* to care about waveform pixels:

- **Weighted reconstruction** (Charbonnier/L1):
  - `L_rec = mean( w * charbonnier(pred, tgt) )`
- **Weighted teacher consistency** (BCE on teacher probabilities):
  - `L_teach = mean( w * BCEWithLogits(stageB(pred), teacher_prob) )`
- **Weighted gradient loss** (optional / often kept very small):
  - `L_grad = mean( w * |∇pred − ∇tgt| )`

Recommended “waveform‑focused” loss (v2):

```
L = 1.0 * L_rec_weighted
  + 1.0 * L_teacher_weighted
  + 0.02 * L_grad_weighted
  + 0.1 * L_identity
```

#### New failure mode to watch: “over‑inking” / black collapse

After weighting, the model may “cheat” the teacher by producing **too much ink** (thicker strokes, extra lines, sometimes **double baselines**) or globally darkening the page. This shows up as:

- `ink_mean(StageB(StageA(src)))` becoming **much larger** than `ink_mean(StageB(tgt))`
- Stage A debug images turning **too dark** or dominated by horizontal lines

Mitigations (apply in this order):

1) **Normalize the weight map** so background is not “free”:
   - `w = w / (mean(w) + 1e-6)` per image, then use `(per_pixel_loss * w).mean()`
2) **Reduce teacher pressure**:
   - lower `beta` in the weighted teacher loss (e.g., `20 → 8–10`) or reduce `w_teach` (e.g., `1.0 → 0.6–0.7`)
3) **Disable gradient loss temporarily** if you see borders/grid/baseline artifacts:
   - set `w_grad = 0.0` for 3–5 epochs, then re‑enable at `0.01`
4) Add a small **photometric anchor** (optional) to prevent global darkening:
   - match per‑image mean/std of `pred` to `tgt` with a small weight (e.g., `0.1–0.2`)
5) If Stage A outputs still collapse after ~20–30 epochs, switch to **crop training** (teacher‑guided waveform crops) so the network can’t win by background dominance.

#### Diagnostics we rely on now

- **Debug snapshots** each epoch (src / tgt / pred, plus teacher overlay).
- `ink_mean`:  
  - `ink_mean_tgt = mean(max_k sigmoid(StageB(tgt)))`  
  - `ink_mean_pred = mean(max_k sigmoid(StageB(StageA(src))))`  
  Healthy training usually has `ink_mean_pred` in the same ballpark as `ink_mean_tgt` (not orders of magnitude smaller or larger).

### 4.6 Training recipe (starter)
- Optimizer: AdamW
- LR: 2e-4
- Weight decay: 1e-4
- AMP: ON
- Batch size: 8 (reduce if VRAM limited)
- Epochs: start 10, extend if val keeps improving
- Optional: cache canonicalized images to `.npy` for speed

## 5) Integration into Inference

Current inference (Stage B only):
- `g = preprocess_for_model(...)` (uint8 gray)
- Stage B → logits/masks → decode → submission

New inference (Stage A + Stage B):
1) `g = preprocess_for_model(...)` (uint8 gray)
2) `g2 = StageA(g)` (cleaned gray, same shape)
3) Stage B on `g2` as usual

Minimal integration (conceptual):

```python
g_u8 = preprocess_for_model(png_path, final_h=H_T, final_w=W_T, rectify=True)

# Stage A expects float in [0,1]
x = torch.from_numpy(g_u8.astype("float32")/255.0)[None,None].to(device)
with torch.no_grad():
    y = torch.sigmoid(stageA(x)).clamp(0,1)

g2_u8 = (y[0,0].cpu().numpy()*255).astype("uint8")

# Stage B runs as before using g2_u8 (or convert to float similarly)
logits = stageB(torch.from_numpy(g2_u8.astype("float32")/255.0)[None,None].to(device))
```

## 9) TL;DR

- Stage B works on clean canonical scans but fails on hard domains.
- Deterministic preprocessing is not enough.
- Add Stage A: a scanifier trained to map hard domains → clean canonical `0001`.
- Train Stage A with a **teacher-consistency loss** using frozen Stage B.
- Inference becomes: preprocess → StageA → StageB → decode → submit.
